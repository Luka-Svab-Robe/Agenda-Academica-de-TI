Agenda Acadêmica de TI

Projeto Acadêmico | Universidade São Judas

Este repositório contém a documentação e o protótipo de um projeto de Especificação de Software desenvolvido no 1º semestre do curso de Análise e Desenvolvimento de Sistemas.

1. Visão Geral do Projeto

O projeto "Agenda Acadêmica para Alunos de TI" é um sistema web focado em auxiliar estudantes de graduação na área de Tecnologia da Informação a gerenciar suas vidas acadêmicas.

O objetivo principal é criar uma ferramenta que vá além de uma simples agenda de tarefas. O sistema permite não apenas o gerenciamento de disciplinas e atividades, mas também oferece um diferencial chave: um módulo para cálculo de médias e desempenho baseado nos pesos e notas lançados pelo próprio aluno.

2. Funcionalidades Principais

O sistema foi projetado para incluir os seguintes requisitos funcionais:

Gestão de Usuário: Cadastro e login de alunos.

Gestão de Disciplinas: Permite ao aluno cadastrar as disciplinas que está cursando.

Gestão de Atividades:

Adicionar atividades (provas, trabalhos, tarefas) vinculadas a uma disciplina.

Definir data de entrega e nível de prioridade (Baixa, Média, Alta).

Marcar atividades como "Concluídas".

Dashboard: Uma tela inicial que exibe as próximas atividades a vencer, facilitando a organização.

Calculadora de Média (Diferencial):

Permite ao aluno definir os pesos de cada avaliação (Ex: P1 - 40%, Trabalhos - 60%).

Lançar as notas obtidas em cada atividade.

O sistema calcula e exibe automaticamente a média parcial do aluno na disciplina.

3. Tecnologias Utilizadas

Este projeto é um protótipo não funcional (interativo) desenvolvido com tecnologias web padrões:

HTML5: Para a estruturação do conteúdo.

Tailwind CSS: Para a estilização e criação de uma interface moderna e responsiva.

JavaScript: Para a interatividade das abas e a simulação dos protótipos de tela.

4. O Documento de Especificação

O arquivo Especificação de Software.html neste repositório é a entrega central do projeto. Ele é um documento interativo que detalha:

Introdução: O escopo e os objetivos do projeto.

Requisitos: A listagem completa de Requisitos Funcionais (RF) e Não Funcionais (RNF).

Protótipos: Uma simulação interativa (clicável) das principais telas do aplicativo (Login, Dashboard, Detalhes da Disciplina).

Modelagem de Dados: O Modelo Entidade-Relacionamento (MER) e o Modelo Lógico (esquema de tabelas) pensados para o sistema.

Referências: A base bibliográfica utilizada para a concepção do projeto.

Este é um projeto acadêmico e serve como uma demonstração de conceitos de Engenharia de Software e Modelagem de Dados.
